# TUI publishing pipeline + user-facing docs

Status: design approved 2026-04-28.

## Goal

Make the `adblocker-tui` Rust binary installable in one shell command on
Linux, and make `docs/TUI.md` a real user manual that someone with no
project context can follow.

## Background

Today the TUI ships only as source. `tui/` is a Cargo project; the
existing `.github/workflows/release.yml` builds a TUI binary on tag for
four targets but only attaches the tarball to the GitHub Release — no
installer script, no auto-update path. `docs/TUI.md` is design-flavored
prose with an ASCII-art mock-up; it doesn't tell a user how to install,
where the binary goes, or how to use `--ssh`.

## Non-goals

- macOS or Windows TUI builds. The TUI is Linux-only by user decision;
  cross-platform is out of scope.
- Crates.io publish. The package name (`adblocker-tui`) is generic
  enough to collide and we have no demand for cargo-side install.
- Homebrew, npm, or msi installers. Shell-only is sufficient.
- Replacing the Go release pipeline. Only the TUI side changes.

## Architecture

Two artifacts:

1. **`docs/TUI.md`** — user manual: what it is, install, run, keys,
   update, troubleshoot. Preserves a one-paragraph "how it works"
   pointer to the architecture spec.
2. **cargo-dist publishing** — the TUI gets its own release pipeline
   (`.github/workflows/release.yml`, generated by cargo-dist). The
   existing `release.yml` is renamed to `release-go.yml` with the
   `rust-tui` job deleted. Both pipelines fire on `v*` tags; their
   asset names don't collide.

## Component 1 — cargo-dist config

**`tui/Cargo.toml`** gains the publish metadata and the `[profile.dist]`
block (mirrors the e2s repo):

```toml
[package]
name        = "adblocker-tui"
version     = "0.1.0"
edition     = "2021"
description = "Ratatui-based TUI for the eBPF adblocker (manage blocklist, allowlist, stats, events)"
license     = "MIT"
repository  = "https://github.com/sandeshgrangdan/adslicer-3000-ebpf-carnage"
homepage    = "https://github.com/sandeshgrangdan/adslicer-3000-ebpf-carnage"

# ...existing [[bin]] and [dependencies]...

[profile.dist]
inherits = "release"
lto      = "thin"
```

**`dist-workspace.toml`** at repo root:

```toml
[workspace]
members = ["cargo:tui"]

[dist]
cargo-dist-version = "0.30.3"
ci                 = "github"
installers         = ["shell"]
targets = [
  "x86_64-unknown-linux-gnu",
  "aarch64-unknown-linux-gnu",
  "x86_64-unknown-linux-musl",
]
unix-archive       = ".tar.gz"
install-path       = "CARGO_HOME"
install-updater    = true
```

`members = ["cargo:tui"]` scopes cargo-dist to the Rust crate; the Go
side of the repo is invisible to it.

## Component 2 — Workflow split

Rename and edit:

- `release.yml` → `release-go.yml`. Delete the `rust-tui` matrix job
  (lines 73–152 in the current file). Keep all other jobs (Go binary
  build, deb, rpm, GHCR image) verbatim.
- `cargo dist init --yes` writes a fresh `release.yml` that publishes
  the TUI on tag. The generated workflow is committed to git;
  re-running `cargo dist init` regenerates it from
  `dist-workspace.toml` if config changes.

Tag form stays `v*`. Both workflows fire on the same tag push.

Asset names for v0.1.0:

- Go side: `adblockerctl_0.1.0_linux_amd64.tar.gz`, `*.deb`, `*.rpm`,
  `ghcr.io/sandeshgrangdan/adblocker:0.1.0`.
- TUI side: `adblocker-tui-x86_64-unknown-linux-gnu.tar.gz`,
  `adblocker-tui-aarch64-unknown-linux-gnu.tar.gz`,
  `adblocker-tui-x86_64-unknown-linux-musl.tar.gz`,
  `adblocker-tui-installer.sh`.

No name collisions.

## Component 3 — `docs/TUI.md` rewrite

Replaces the current file. Section order:

1. **What is `adblocker-tui`** — one paragraph framing.
2. **Install** — three options:
   - (a) curl one-liner pointing at the cargo-dist installer URL on the
     latest GitHub Release.
   - (b) prebuilt tarball download (`adblocker-tui-<target>.tar.gz` from
     Releases) + `install -m 0755`.
   - (c) `cargo install --path tui` from a source clone.
3. **Run it** — local with sudo, local with `--no-sudo`, remote with
   `--ssh user@host`. One example each.
4. **Keybindings** — single table covering `?`, `Tab` / `Shift+Tab`,
   `1`–`4`, `j` / `k` / arrows, `a`, `t`, `d`, `r`, `u`, `q`.
5. **Updating** — `adblocker-tui --update` (provided by
   `install-updater = true`).
6. **Troubleshooting** — four concrete failures with copy-paste fixes:
   pinned map missing (daemon not running),
   `sudo: a password is required` (passwordless sudo / polkit),
   `Permission denied (publickey)` over `--ssh` (BatchMode + key auth),
   `command not found: adblockerctl` on the daemon side (PATH).
7. **How it works (brief)** — one paragraph + link to
   `docs/superpowers/specs/2026-04-28-tui-architecture.md`.

The current `docs/TUI.md` design content (ASCII mockup, "how the TUI
talks to adblockerctl", etc.) moves verbatim to a new file
`docs/superpowers/specs/2026-04-28-tui-architecture.md` so the
architectural detail isn't lost.

## Release process (operator-facing)

```sh
# 1. bump tui/Cargo.toml `version`
# 2. (only if dist-workspace.toml changed) cargo dist init --yes
# 3. git tag v0.1.0
# 4. git push origin v0.1.0
```

Both `release-go.yml` and the generated `release.yml` run; ~5 minutes
later the v0.1.0 GitHub Release has TUI tarballs, an
`adblocker-tui-installer.sh`, and the Go-side artifacts. Users install
with:

```sh
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/sandeshgrangdan/adslicer-3000-ebpf-carnage/releases/download/v0.1.0/adblocker-tui-installer.sh | sh
```
